## 객체지향 프로그래밍 (OOP)

프로그램을 순차적인 명령어의 목록이 아닌 객체들의 모임으로 파악하고 이들 간의 상호작용으로 로직을 구성하는 프로그래밍

**객체** 

속성(데이터)과 행위(메서드)를 하나로 묶어 가지는 실체

**목적**
1. 모듈화 : 객체단위로 쪼개져 있어서 레고 블록처럼 조립하고 분리하기 쉬움
2. 재사용성 : 잘 만든 객체는 다른 프로그램에서 사용이 가능
3. 유지보수성 : 수정이 필요할 경우 필요한 부분(객체)만 수정이 가능


**디자인 패턴이란 ?**

개발중에 자주 발생하는 문제에 대한 해결 방식




## 싱글톤

**전체 프로그램에서 단한의 객체만 존재하도록 보장하는 패턴**

현실에서 예를 들면 집에서 pc가 한대 밖에 없어서 가족 모두가 하나의 pc를 공유해서 사용

**사용목적**
1. 메모리 낭비를 방지 : 고정된 메모리 영역을 사용으로 메모리 절약
2. 데이터를 공유 : 전역 인스턴스이기 때문에 다른 클래스간에 데이터를 공유 쉬움

**예시코드**
```python
class Sigleton:
    _instance = None # 유일한 인스턴스
    
    def __new__(cls):
        # 인스턴스가 아직 없다면 생성
        if cls._instance is None:
            print("--- 새로운 인스턴스 생성 ---")
            cls._instance = super(Singleton,cls).__new(cls)
        # 이미 존재 한다면
        else:
            print("--- 이미 있는 인스턴스를 반환 ---")

        return cls.instance
# 실제 사용
# 첫번째 생성 시도
s1 = Singleton()
# 두번째 생성
s2 = Singleton()

# 두 변수가 가리키는 객체가 같은지 확인
print(f"s1와 s2는 같은 객체인가 {s1 is s2}")
